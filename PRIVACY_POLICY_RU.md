## 2. ОПРЕДЕЛЕНИЯ

**Персональные данные** — любая информация, относящаяся к прямо или косвенно определенному или определяемому физическому лицу (субъекту персональных данных).

**Обработка персональных данных** — любое действие или совокупность действий с персональными данными, включая сбор, запись, систематизацию, накопление, хранение, уточнение (обновление, изменение), извлечение, использование, передачу (распространение, предоставление, доступ), обезличивание, блокирование, удаление, уничтожение персональных данных.

**Оператор** — организация или физическое лицо, самостоятельно или совместно с другими лицами организующие обработку персональных данных, а также определяющие цели обработки персональных данных.

**Субъект персональных данных** — пользователь Приложения, физическое лицо, к которому относятся персональные данные.

## 3. КАТЕГОРИИ ОБРАБАТЫВАЕМЫХ ПЕРСОНАЛЬНЫХ ДАННЫХ

Приложение обрабатывает следующие категории персональных данных:

### 3.1. Данные, необходимые для регистрации и использования Приложения (обязательные):

- **Номер телефона** — используется для регистрации, авторизации и восстановления доступа к аккаунту
- **Имя и фамилия** — для идентификации пользователя в Приложении
- **Дата рождения** — для расчета возраста и персонализации контента
- **Пол** — для статистики и персонализации
- **Город проживания** — для отображения в профиле и поиска пользователей

### 3.2. Данные профиля пользователя (опциональные):

- **Никнейм** — публичное имя пользователя
- **Email** — для связи и восстановления доступа (если указан)
- **Фотографии профиля** — аватар и фоновое изображение
- **Физические параметры:**
  - Рост
  - Вес
  - Максимальный пульс (ЧСС)
- **Основной вид спорта** — бег, велосипед, плавание, лыжи

### 3.3. Данные о местоположении:

- **GPS-координаты маршрутов тренировок** — широта, долгота, высота над уровнем моря, временные метки
- Данные о местоположении собираются только во время активных тренировок и используются исключительно для отображения маршрута на карте
- Данные хранятся локально на устройстве (SQLite) и на сервере для синхронизации между устройствами

### 3.4. Данные о здоровье и тренировках:

- **Данные из Health Connect (Android) и HealthKit (iOS):**
  - Пульс (частота сердечных сокращений)
  - Дистанция тренировки
  - Длительность тренировки
  - Сожженные калории (активные и общие)
  - Количество шагов
  - Данные о маршруте тренировки (GPS-координаты)
- **Статистика тренировок:**
  - Тип активности (бег, велосипед, плавание, лыжи)
  - Дистанция, скорость, время
  - Высота над уровнем моря
  - Используемое снаряжение

**Важно:** Доступ к данным о здоровье предоставляется только с вашего явного согласия через системные диалоги Android Health Connect или iOS HealthKit. Вы можете в любой момент отозвать доступ в настройках устройства.

### 3.5. Медиа-контент:

- **Фотографии тренировок** — загружаемые пользователем изображения
- **Видео тренировок** — загружаемые пользователем видеозаписи

### 3.6. Социальные данные:

- **Лайки** — информация о том, какие посты пользователь отметил как понравившиеся
- **Комментарии** — текстовые комментарии к постам других пользователей
- **Подписки и подписчики** — список пользователей, на которых вы подписаны, и которые подписаны на вас
- **Публичные посты** — информация о тренировках, размещенная пользователем в публичной ленте

### 3.7. Технические данные:

- **FCM токены** — идентификаторы для отправки push-уведомлений
- **Device ID** — уникальный идентификатор устройства (Android ID для Android, IDFV для iOS)
- **Тип устройства** — Android или iOS
- **Данные кэширования** — локальное хранение данных для работы Приложения без интернета (SQLite)

### 3.8. Данные контактов:

- **Доступ к телефонной книге** — используется исключительно для поиска друзей по номеру телефона из вашей телефонной книги
- Контакты не загружаются на сервер и не передаются третьим лицам
- Доступ к контактам запрашивается только с вашего явного согласия

   - Анализ использования Приложения для улучшения функционала

## 6. СПОСОБЫ И СРОКИ ОБРАБОТКИ ПЕРСОНАЛЬНЫХ ДАННЫХ

### 6.1. Способы обработки:

Обработка персональных данных осуществляется с использованием средств автоматизации и без использования таких средств.

Данные обрабатываются:
- **На сервере Оператора** — для синхронизации между устройствами и обеспечения работы социальных функций
- **Локально на устройстве пользователя** — для быстрой работы Приложения и возможности использования без интернета (кэширование в SQLite)

### 6.2. Сроки обработки:

- Персональные данные обрабатываются в течение всего срока использования Приложения пользователем
- После удаления аккаунта персональные данные удаляются в течение 30 дней, за исключением случаев, когда хранение данных требуется в соответствии с законодательством
- Данные, необходимые для выполнения требований законодательства (например, для налогового учета), могут храниться в течение сроков, установленных законодательством

### 6.3. Удаление данных:

Пользователь может в любой момент удалить свой аккаунт через настройки Приложения. После удаления аккаунта:
- Профиль пользователя становится недоступным для других пользователей
- Персональные данные удаляются с сервера в течение 30 дней
- Локальные данные на устройстве могут быть удалены при удалении Приложения

**Исключения:** Некоторые данные могут сохраняться в течение более длительного срока, если это необходимо для:
- Выполнения требований законодательства
- Разрешения споров
- Предотвращения мошенничества или злоупотреблений

## 9. ПЕРЕДАЧА ПЕРСОНАЛЬНЫХ ДАННЫХ ТРЕТЬИМ ЛИЦАМ

Оператор может передавать персональные данные третьим лицам только в следующих случаях:

### 9.1. Технические сервисы:

Для обеспечения работы Приложения используются следующие технические сервисы третьих сторон:

- **Firebase Cloud Messaging (Google)** — для отправки push-уведомлений. Передаются только FCM токены и данные устройства, необходимые для доставки уведомлений
- **MapBox** — для отображения карт. Передаются только координаты маршрутов для отображения на карте
- **Health Connect (Google, Android)** и **HealthKit (Apple, iOS)** — для получения данных о здоровье и тренировках. Доступ предоставляется только с вашего явного согласия через системные диалоги

### 9.2. Условия передачи:

- Передача данных третьим лицам осуществляется только в объеме, необходимом для предоставления соответствующих сервисов
- Третьи лица обязаны обеспечивать конфиденциальность переданных данных в соответствии с их политиками конфиденциальности
- Оператор не продает и не передает персональные данные третьим лицам для маркетинговых целей

### 9.3. Требования законодательства:

Оператор может передавать персональные данные по запросам государственных органов в случаях, предусмотренных законодательством Российской Федерации.
